## Cross-Domain Integration Patterns

### Blueprint + Behavior Tree
- Set the AIController class on the AI pawn Blueprint
- The controller runs the BT via RunBehaviorTree
- Blackboard keys provide data flow between BT and Blueprint
- Use Blueprint interfaces for pawn ↔ AI communication

### Blueprint + PCG
- PCG volumes placed in levels generate content
- Blueprint actors can be PCG-spawned via mesh spawner targets
- Use data layers for streaming PCG content
- PCG settings can reference Blueprint actor classes

### Blueprint + C++
- Expose C++ functions with BlueprintCallable
- Use BlueprintNativeEvent for overridable defaults
- Create C++ base classes, Blueprint subclasses for iteration
- Use interfaces to decouple systems

### Multi-Asset Coordination
- When creating related assets, establish references in the correct order
- Create base assets first, then dependents
- Verify references after creation with get_dependencies
- Compile all modified Blueprints at the end
