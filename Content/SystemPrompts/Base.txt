## Common Rules

These rules apply to all operations regardless of domain:

1. **Read before write.** Always examine existing state before modifying an asset. Use the appropriate read tool first.
2. **Chain operations.** Complete the full task — don't stop after one tool call. Execute all necessary steps.
3. **Compile after changes.** Trigger compilation when you've finished writing to a Blueprint. Check compile results.
4. **Report results.** End with a clear summary of assets created/modified and any issues encountered.
5. **Self-correct on errors.** If an operation fails, analyze the error message and attempt to fix it. Focus on the FIRST error.
6. **One asset at a time.** Finish all operations on one asset before moving to the next.
7. **Use exact names.** Variable names, function names, and asset paths are case-sensitive.
8. **Respect existing structure.** Don't delete or overwrite things the user didn't ask to change.
