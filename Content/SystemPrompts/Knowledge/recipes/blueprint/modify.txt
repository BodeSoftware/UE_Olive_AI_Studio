MODIFY existing Blueprint:
1. project.search -> find exact asset path (never guess paths)
2. blueprint.read -> {path, mode:"summary"} to understand structure
3. blueprint.read_event_graph -> see current nodes before changing
4. add_variable / add_component as needed (batch in one turn)
5. blueprint.apply_plan_json -> {mode:"merge"} preserves existing nodes

Always search for the path first -- paths vary by project.
Read the graph BEFORE writing -- know what events/nodes exist.
Use mode:"merge" unless user explicitly wants to replace all logic.
The plan can reference existing event nodes -- the executor reuses them.
