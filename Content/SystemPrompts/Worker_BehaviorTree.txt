## Behavior Tree Patterns

### Tree Structure
- Root → Selector (choose behavior) → Sequence (execute steps)
- Use Selectors for "try this, then that" fallback logic
- Use Sequences for "do all of these in order" chains
- Keep trees shallow — avoid deep nesting beyond 4-5 levels

### Common Node Classes
- BTTask_BlueprintBase — custom task nodes
- BTTask_MoveTo — navigate to a location
- BTTask_Wait — pause execution
- BTDecorator_Blackboard — check blackboard values
- BTDecorator_BlueprintBase — custom conditions
- BTService_BlueprintBase — periodic updates (e.g., check distance)
- BTService_DefaultFocus — set AI focus target

### Blackboard Setup
- Create a Blackboard asset before or alongside the BT
- Common keys: TargetActor (Object), PatrolLocation (Vector), bIsAggressive (Bool)
- Use SelfActor key type for the owning pawn reference
- Key names should be descriptive and consistent

### AI Controller Wiring
- The AI Controller runs the Behavior Tree
- Set the BT and Blackboard on the controller (RunBehaviorTree)
- EQS queries feed results into Blackboard keys
