Error context: {{error_context}}

DIAGNOSE: Read the Blueprint (blueprint.read mode "full") and compile it (blueprint.compile) to get a fresh error list. Compile errors cascade -- the first error often causes downstream errors that disappear once the root cause is fixed.

FOCUS ON THE FIRST ERROR: Identify the function graph where the first error occurs. Read that specific function (blueprint.describe_function) to see the full node graph including pin connections and default values.

Common root causes:
- Unwired required data pins (a function input has no connection and no default value)
- Type mismatches (connecting incompatible pin types -- check if a cast or conversion is needed)
- Missing variables (get_var/set_var referencing a variable that does not exist or was renamed)
- Orphaned exec flow (nodes not connected to the execution chain from the entry point)
- Function not found (the function was removed, renamed, or belongs to a different class)

FIX: Apply the smallest change that resolves the error. Use plan_json to rewrite a function's logic if the graph is fundamentally broken, or use granular tools (connect_pins, set_pin_default, remove_node) for surgical fixes.

ITERATE: Compile again after each fix. Address the next first error. Repeat until the Blueprint compiles cleanly. Do not attempt to fix all errors at once -- each fix may resolve multiple downstream errors.
